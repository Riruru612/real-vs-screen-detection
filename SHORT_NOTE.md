# Short Note

## Spot the Fake Photo – Approach

The objective of this assignment was to develop a lightweight computer vision system capable of distinguishing genuine photographs from photographs of digital screens. Although both image types appear visually similar, screen recaptures often contain subtle artifacts introduced by the display hardware and camera imaging pipeline, including moiré patterns, pixel grid structures, reflections, frequency-domain distortions, and texture inconsistencies.

Rather than relying on a single modeling approach, I treated this task as a model selection problem and systematically compared both classical machine learning and deep learning techniques to identify the most reliable solution.

I first collected a balanced custom dataset consisting of **102 smartphone-captured images (51 real photographs and 51 screen recaptures)** captured under different lighting conditions, viewing angles, backgrounds, and object categories. After performing exploratory data analysis, I extracted handcrafted image descriptors—including Histogram of Oriented Gradients (HOG), Local Binary Patterns (LBP), Fast Fourier Transform (FFT), Laplacian variance, edge density, reflection statistics, and color features—to establish strong classical machine learning baselines. These engineered features were used to train both Random Forest and XGBoost classifiers.

While the handcrafted-feature models produced encouraging results, I expected transfer learning to capture higher-level visual representations that are difficult to engineer manually. Consequently, I evaluated multiple ImageNet-pretrained convolutional neural networks, including **MobileNetV3 Small, EfficientNet-B0, ResNet18, and ResNet34**, using **Stratified 5-Fold Cross Validation**. To ensure a fair comparison, every architecture was trained using the same experimental setup, including transfer learning, AdamW optimization, data augmentation, learning-rate scheduling, and early stopping.

Among all evaluated models, **ResNet34** demonstrated the best overall balance between accuracy, precision, recall, and robustness, achieving a **mean cross-validation accuracy of 85.33%**, a **mean F1-score of 86.06%**, and a **best validation fold accuracy of 95.00%**. Although EfficientNet-B0 achieved a perfect score on one validation fold, its average performance across all folds was lower and less consistent. Given the relatively small dataset, I prioritized consistency across multiple validation splits rather than selecting a model based on a single high-performing fold. After model selection, ResNet34 was retrained using the complete dataset to maximize the available training data before deployment.

The final solution includes a lightweight **PyTorch inference pipeline** (`predict.py`) together with an interactive **Streamlit web application** for demonstration. Inference is performed entirely **on-device**, achieving an average latency of approximately **30–40 ms per image** on an Apple MacBook Air M2 CPU. Since no external APIs or cloud infrastructure are required, the effective inference cost is **approximately $0 per image**, making the solution suitable for lightweight real-time deployment.

## What I Would Improve

The primary limitation of this project is the size and diversity of the dataset. Given additional time, I would expand the dataset by collecting images from multiple smartphone cameras, display technologies (LCD, OLED, AMOLED, and Mini-LED), lighting conditions, viewing angles, and environmental settings to improve the model's ability to generalize to unseen data.

I would also investigate hybrid approaches that combine frequency-domain features with deep visual representations, explore newer architectures such as Vision Transformers, and optimize the model for mobile deployment through quantization and TorchScript or ONNX export. Finally, rather than using a fixed decision threshold of 0.5, I would select the operating threshold using ROC and Precision–Recall analysis based on the desired trade-off between false positives and false negatives. In a production environment, I would continuously collect newly observed fraud examples, perform hard-negative mining, and periodically retrain the model to ensure that detection performance remains robust as fraud patterns evolve.
